「自分の文章を、noteみたいに気軽に更新できる場所がほしい」。
ただし、コメントはいらない。タグもいらない。見た目はちゃんとしていたい。トップとプロフィールも欲しい。
さらに「下書き」は一覧に出したくないけど、URLを知っている自分だけが確認できる形にしたい。

この条件を満たすために、GitHub Pages と Astro のブログテンプレートを使って、個人サイトを立ち上げた。
ここでは、その完成までの流れをまとめる。

## 目標（要件）
今回のゴールは次の通り。

- GitHub Pages で個人サイトを公開する
- ブログも同じサイト内で運用する（noteに近い感覚）
- 見た目はテンプレートで整える（自作デザインは後回し）
- コメント機能は付けない
- タグ機能も付けない
- トップページとプロフィールページを持つ
- 下書き（draft）は一覧やRSSに出さない
- ただし、下書きはURL直打ちなら表示できる（「知っている人だけ見える」運用）

## 採用した構成
最終的に、次の構成を選んだ。

- 公開は GitHub Pages（ユーザーサイト）
- テンプレートは Astro の blog starter 系
- デプロイは GitHub Actions（pushしたら自動公開）
- 記事は Markdown（ブラウザでも編集しやすい）
- 下書きは frontmatter に draft: true を持たせる

「ブラウザで下書きしたい」という要望に対して、Markdownを GitHub のWeb画面で直接編集する運用が一番現実的だった。
見た目はテンプレートが最初から担保してくれる。

## まずはサイトの公開まで到達する
最初の到達点は単純で、「テンプレートのトップページが公開URLで表示されること」。

これを満たすためにやったことは以下。

- ユーザーサイト用のリポジトリを作る（GitHub Pagesに対応した名前）
- Astroテンプレートを入れる
- Pages の Source を GitHub Actions にする
- Actions のデプロイが成功して、公開URLが表示できる状態にする

この時点で、「見た目が整ったサイトが、ネット上で見える」までは完成する。

## ローカルでも確認できるようにする（npm / build）
公開ができたあと、手元でも build を通して確認できる状態にしておくと、変更が怖くなくなる。

基本はこの2つ。

- 依存関係のインストール（初回のみ）
- ビルド実行

手順はシンプル。

- npm ci（または npm install）
- npm run build

これでビルドが通れば、コードや構成の大枠は破綻していない。

## 目玉：下書き（draft）運用を入れる
今回の核心はここだった。

下書きの方針は「一覧とRSSに出さない。でもURL直打ちなら表示できる」。

この方針の良い点は、運用がとにかく軽いこと。
記事の先頭に draft: true を入れるだけでいい。

実装としてやったことは、2か所だけ。

1) ブログ一覧ページ  
ブログ一覧を作っているページで、記事取得時に draft を除外する。

2) RSS  
RSS生成で記事を集める時にも、draft を除外する。

逆に、「記事ページ側」は除外しない。
これが「URL直打ちなら表示できる」を成立させるポイントになる。

## 記事ファイルの置き場所を統一する
Astroテンプレートには大きく2つの置き場がある。

- src/pages/：ページそのもの（ここに置くと“ページ”として扱われる）
- src/content/：ブログ記事の“コレクション”（ここに置くと一覧生成などの仕組みに乗る）

今回の運用（一覧で管理しつつ、draftフィルタも効かせたい）では、記事は src/content/blog/ に置くのが正解だった。
下書き記事もここに置く形に統一した。

## 下書き記事の書き方（テンプレ運用）
src/content/blog/ 配下に md（または mdx）を作り、frontmatter に draft を入れる。

例：

---
title: "下書きテスト"
pubDate: 2026-01-03
description: "非公開の下書き"
draft: true
heroImage: "/blog-placeholder-1.jpg"
---

挙動はこうなる。

- ブログ一覧に出ない
- RSSに出ない
- ただし、URL直打ちなら表示できる

この「軽さ」が、noteっぽい運用感に近い。

## ここまでで達成したこと
この時点で、当初の要件は満たせている。

- 見た目が整った個人サイトを公開できた
- ブログも同じサイト内で運用できる
- コメントやタグに依存しない構成
- トップページとプロフィールページがある（テンプレートに含まれる）
- draft運用ができる（一覧とRSSには出ない／URL直打ちはOK）
- ローカルで npm ci → npm run build も通る

## 次にやると良さそうなこと（任意）
ここから先は「好み」の領域。

- トップを“noteっぽい導線”にする（最新記事を数件だけ出す、カードを大きくする等）
- プロフィールページを整える（固定文、リンク、簡単な経歴など）
- 下書きを検索エンジンに拾わせない（noindexの付与）
  ※ URL直打ち運用と両立はできる
