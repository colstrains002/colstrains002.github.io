GitHub Pages（Astro）のブログを、テンプレ状態のまま放置していたのがずっと気になっていて、年始のタイミングで「自分用に最低限まともにする」と決めて触り始めた。最終的には、記事を書く作業も含めて、なるべくブラウザ内で完結できる運用に寄せたい。いまはその手前の「土台づくり」をひたすらやっていた感じです。

やりたいことは最初から割と明確で、テンプレのサンプル記事を消して、トップページとAboutを自分の内容に差し替えて、下書き（draft）の扱いだけは崩さない、というもの。draftは「直リンクなら見えていいけど、一覧（/ と /blog）とRSSには出さない」を維持したい。ここが崩れると困るので、まずはここを固める方針にした。

最初にやったのはテンプレ記事の掃除。サンプルの first-post みたいなものや、スタイルガイド、MDXの例など、テンプレのにぎやかしが残っていたので、不要なものは削除して整理した。残すものは自分の記事と下書きだけ。ここが片付くと、ようやく「自分の場所」になってきた気がします。

次に、draft運用が本当に意図どおりになっているかを確認した。一覧とRSSは draft を除外し、記事詳細側では draft を理由に表示を止めない。つまり「一覧とRSSでは隠す、直リンクはOK」という形に揃える、ということです。ここはサイト運用の根っこなので、先に責務を確認して、なるべく揺れにくい形に寄せた。

そのうえで、トップページとAboutページを自分用に差し替えた。トップは短い紹介文と最新記事数件、ブログ一覧へのリンク、という最小構成にして、当然ここでも draft は除外する。Aboutはテンプレ文言を全部捨てて、必要最小限のプロフィールとリンクだけにした。見た目の凝り方は後回しで、まずは変なテンプレ文が残っていない状態にしたかったので、いまはこれで十分だと思っています。

サイトタイトルもテンプレのままだったので、自分の名前（Colstrains）に変更した。タイトルはヘッダーだけでなく、タブ表示やRSSにも影響するので、参照元を一本化してまとめて揃えるのが良さそうだ。ついでにサイト説明文も、短くて使いやすい文に整えて、タブやOGP、RSSに出ても違和感が出にくい形にした。日本語＋英語を1本にする案もあったが、長く見えることもありそうなので、表示の見え方を見つつ調整する前提にしている。

途中、ビルドやコミットが通らない瞬間があって少し焦った。原因はだいたい「実行している場所が違う」とか「いま見ているものが違う」とか、その手の単純なことのようだった。エラーメッセージは割と正直なので、まず前提を疑って一つずつ戻る。こういうのはまた起きそうなので、手順として覚えておきたい。

テンプレの残骸でいちばん目につくのが、右上のSNSアイコンとフッターの著作権表示だった。SNSリンクは Astro公式になっていたので、自分のものに差し替える。著作権表示も「Your name here」になっていたので、年とサイト名を使う形に直して整えた。

SNSアイコンについては、テンプレ内に素材がまとまって入っているのでは、と一瞬疑った。アイコン用の属性が残っていたからだ。でも実際に探してみると、独立した素材としてはほぼ見当たらず、HeaderやFooterにインラインで埋め込まれているだけだった。なので方針を変えて、アイコンは公開ディレクトリに置いて参照する形に統一した。こうしておけば、ヘッダーとフッターの両方で同じアイコンを使えるし、差し替えも一か所で済む。

当初は Pixiv / X / note にしたかったのだが、Pixivの公式SVGが白と青しかなくて、白背景に白は見えないし、青にすると他と色がずれて見えそうだった。悩んだ末、Pixiv枠はInstagramに切り替えることにした。まずリンクを揃えるのが先で、見た目の完璧さは後で詰めればいい、という判断。結果として、ヘッダーとフッターのSNSは Instagram / X / note に統一できて、表示もリンクも確認できた。これで最低限の「テンプレ状態から脱出」はできたと思う。

本題。次は「ブログ記事を書くとき、ブラウザ内で完結したい」をちゃんと形にしたい。普段はブラウザ上のエディタで記事を書いてコミットし、Actionsで公開、で回せそうだし、必要なときだけクラウド開発環境を使えば、ビルド確認やプレビューもブラウザで完結できそうだ。どこまでを日常運用にするか、最小のテンプレ（frontmatter）と手順を固定していきたいです。

今日はここまで。余力のあるときに、ブラウザ完結の「記事作成テンプレ」と「公開切替（draft true→false）」の型を決めて、次の記事を実際にその手順で出してみます。
